## 2. 변경 배경

프로젝트의 1차 목표는 `prompt`를 제대로 만들어 Claude Code/Roo Code가 실제 Python package 구현을 진행할 수 있게 하는 것이다.

단, 사람이 모든 세부 구현 방식을 고정하면 실제 PC 환경, MCP 연결 상태, package manager, 기존 script/skill 사용 가능 여부에 따라 구현이 깨질 수 있다.

따라서 다음 역할 분리가 필요하다.

```text
GPT + 사용자
= 큰 틀 결정

Roo Code / Claude Code
= 구현 환경 조사 + 세부 구현 방식 최적화
```

## 3. 신규 공통 원칙

### 3.1 GPT + 사용자 담당 영역

```text
- 자동화 목적
- 업무 절차
- 기본 아키텍처 방향
- 기본 입력값 / 출력값
- 기본 schema 방향
- Proposal 방향
- 성공 기준
- dry-run 기준
- 다른 5.x 항목과의 역할 경계
```

### 3.2 Roo Code / Claude Code 담당 영역

```text
- OS / shell / workspace 확인
- Python version 확인
- package manager 확인
- repository 구조 확인
- git / Perforce 사용 가능 여부 확인
- Jira / Confluence / Perforce MCP 연결 가능 여부 확인
- .env / config 파일 존재 여부 확인
- 실행 가능한 CLI 도구 확인
- 기존 package / skill / prompt / schema 재사용 가능 여부 확인
- dependency 선택
- schema 세부 필드 조정
- package 구조 세부화
- 테스트 실행 방식 결정
- dry-run 실행 가능 범위 결정
- 구현 중 제약사항과 prompt 개선점 기록
```
