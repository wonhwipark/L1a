## A. 지식 누적 구조 (★최우선 — 6/20 핵심 확인 항목)
# 5.5의 skills_seed / case 누적과 중복되는지가 여기서 갈림.

A1. L1SW의 "지식 누적"은 정확히 어디에, 어떤 형식으로 저장되는가?
    - skill 파일(.md) 자체를 수정하는 방식인가?
    - 별도 데이터 파일(JSON/YAML/DB)에 쌓는가?
    - 아니면 manifest fragment JSON을 갱신하는가?

A2. 누적 단위는 무엇인가?
    - 모듈 기반인가? (예: [DSSM] 모듈에 대한 패턴이 쌓임)
    - 이슈/증상 패턴 기반인가? (예: "msg2 timeout" 같은 현상 단위)
    - 아니면 사람이 수동으로 skill에 메모를 추가하는 구조인가?

A3. 누적된 지식이 다음 분석에 어떻게 재사용되는가?
    - 자동으로 프롬프트/컨텍스트에 주입되는가?
    - 아니면 분석가가 수동으로 참조하는가?

# → A1~A3 답에 따라:
#   "모듈 기반 + 수동" 이면 → 5.5 skills_seed(issue_type 기반)와 안 겹침 → 현재 설계 유지
#   "이슈 패턴 기반 + 자동" 이면 → 5.5 ⑥ 영역 중복 위험 → skills_seed 역할 재정의 필요
