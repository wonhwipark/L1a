## 4. 보류 사유

topic06 Delta는 5.5 RCA Knowledge Graph의 대용량 로그 처리 방향을 잘 정리했지만, 현재 단계에서는 실제 Master/Prompt 반영보다 설계 기록으로 유지하는 것이 적절하다고 판단했다.

주요 이유:

1. 현재 `last_status.md` 기준 topic05 S9 정리 흐름이 아직 남아 있다.
2. topic06은 5.5의 신규 확장 주제이므로 즉시 병합보다 별도 검토 여지를 남기는 편이 안전하다.
3. 500회 이상 반복 분석을 위한 파일 기반 KG 운영안은 유용하지만, 실제 반영 전에 팀 운영 방식과 저장 위치를 추가 논의할 필요가 있다.
4. Step 1 pre-filter는 즉시 실행 가능한 아이디어이나, 실제 script/package 반영 전 환경 기준(PowerShell/Python/Git Bash)을 확정해야 한다.

## 5. 유지되는 설계 자산

보류 처리하더라도 아래 설계 자산은 topic06 Delta에 보존된다.

- 100MB~1GB 로그를 AI Context에 직접 넣지 않는 원칙
- 문제 유형별 pre-filter rule 초안
- `signal_<issue_type>.txt` 생성 흐름
- `RCA_<date>_<issue_type>.md` 정형 Markdown 템플릿
- 500회 이상 반복 분석을 위한 파일 기반 Knowledge Graph 운영안
- `rca_kg/cases`, `signals`, `skills_seed`, `indexes`, `schema` 구조
- case YAML 예시
- index YAML 예시
- skill seed 운영안
- Phase 1~5 확장 로드맵
