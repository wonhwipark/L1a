### 1.3 "간단 모듈소개" 포맷 정의

**배경:**  
Master 5.2에서 "간단한 모듈 설명 문서"를 생성한다고 명시하지만, 구체적 포맷이 미정.

**S4 제안:**

```markdown
포맷 (각 모듈당):
- 분량: 50~100줄 (Markdown)
- 구성:
  1. 모듈 개요 (2~3줄): 모듈의 역할 한 문단
  2. 주요 책임 (bullet 2~3개): "REQ 처리", "CNF 응답", "타이밍 관리" 등
  3. 주요 함수/API (리스트): 함수명 + 1줄 설명
  4. 주요 IPC 호출 (옵션): REQ/CNF 구분, 호출 순서

예시:
```
### TxSwitchMngr (Transmit Switch Manager)

**개요**: Transmit 패스에서 신호 경로를 선택하고 전환하는 모듈.

**주요 책임**:
- TX 신호 경로 선택 (Main/Diversity)
- Path switching 타이밍 관리
- REQ/CNF 메시지 처리

**주요 함수**:
- `TxSwitch_SetPath()`: 신호 경로 설정
- `TxSwitch_GetCurrentPath()`: 현재 경로 조회
- `TxSwitch_HandleReq()`: REQ 메시지 처리

**IPC 호출**:
- REQ 송신: TxCfgMngr 요청 (config 변경 필요 시)
- CNF 수신: 응답 처리
```

**선택지:**
